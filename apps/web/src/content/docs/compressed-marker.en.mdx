# `[DISTILL:COMPRESSED]` marker and PreCompact hook

Shipped in `v0.10.0`. **Opt-in.** Lets Claude Code preserve Distill-compressed text intact during autocompact — instead of summarising it a second time and losing the token-savings work.

## Why this marker exists

Claude Code triggers autocompact automatically when the context window approaches saturation. The summariser LLM reads the full transcript and produces a condensed version. Without an explicit signal, it treats text already compressed by `auto_optimize` / `smart_file_read` / `code_execute` like any other content — and re-summarises it, often dropping details Distill intentionally preserved.

The marker is a **plain-text envelope** wrapped around each compressed payload. The PreCompact hook passes an explicit instruction to the summariser LLM: *"anything wrapped in `[DISTILL:COMPRESSED …]` must be copied verbatim, not summarised"*. This is the only channel that survives autocompact — no MCP field (`structuredContent`, `mcpMeta`) is transmitted to the Anthropic API (see the "Claude Code Mechanics" appendix in `CLAUDE.md`, row documenting `mcpMeta` excluded from API blocks).

## Marker format

```text
[DISTILL:COMPRESSED ratio=X.XX method=<name>]
<compressed payload>
[/DISTILL:COMPRESSED]
```

- **`ratio`**: compressed size / original size, clamped to `[0, 1]`, always two decimals (e.g. `0.23` for 77% savings).
- **`method`**: compressor or mode that produced the payload. Currently emitted values: `auto`, `logs`, `diff`, `semantic`, `skeleton`, `extract`, `search`, `build+recompressed`. Sanitised to `[A-Za-z0-9+_.-]` by `packages/mcp-server/src/utils/distill-marker.ts`.

### Collision escape

If the payload already contains the literal substring `[DISTILL:COMPRESSED` or `[/DISTILL:COMPRESSED]` (for instance because a user is manipulating text that discusses the contract itself), the wrapper falls back to escape tokens:

```text
[DISTILL-USER-TEXT:COMPRESSED ratio=X.XX method=<name>]
<payload>
[/DISTILL-USER-TEXT:COMPRESSED]
```

Consumers can thus tell user-authored text apart from a real Distill marker.

## Enabling

One environment variable:

```bash
DISTILL_COMPRESSED_MARKERS=1
```

Accepted values (case-insensitive): `1`, `true`, `yes`. Default: off — output is identical to `v0.9.x`, no regression on existing integrations.

In `.claude/settings.local.json`:

```json
{
  "mcpServers": {
    "distill": {
      "command": "npx",
      "args": ["distill-mcp"],
      "env": {
        "DISTILL_COMPRESSED_MARKERS": "1"
      }
    }
  }
}
```

## Per-tool thresholds

The wrapper activates only when the compression gain justifies the envelope overhead — otherwise the raw text returns unwrapped (no half-envelope).

| Tool | Wrap condition |
|------|----------------|
| `auto_optimize` | `compressionRatio ≤ 0.7` (≥ 30% savings). Short-input pass-through and the error branch never wrap. |
| `smart_file_read` | `text.length < 0.5 × originalFileLength` **and** mode ∈ `{skeleton, extract, search}`. `full` and `lines` modes never wrap. Wrap applied in both the `cacheAndReturn` path and the cache-hit path. |
| `code_execute` | Through the `ctx.compress.*` helpers (`compressAuto`, `compressLogs`, `compressDiff`, `compressSemantic`). Same ≥ 30% savings rule. |

## PreCompact hook

### Shipped script

Distill ships a turnkey POSIX hook at `packages/mcp-server/scripts/precompact-hook.sh`. It reads Claude Code's `PreCompactHookInput` payload from stdin and emits a plain-text instruction on stdout that Claude Code merges into `newCustomInstructions` (`claude-code/utils/hooks.ts:3991-4024`).

The instruction tells the summariser LLM:

> *Preserve any region wrapped in `[DISTILL:COMPRESSED …][/DISTILL:COMPRESSED]` verbatim. Do not re-summarise, re-format, or paraphrase its contents.*

The script exits with code `0` on **every** input shape — including malformed JSON or unexpected events — so it can never block compaction. `--help` prints the full contract.

### Installation

```bash
# Idempotent install
distill-mcp setup --install-precompact-hook

# Preview changes
distill-mcp setup --install-precompact-hook --dry-run

# Uninstall
distill-mcp setup --uninstall-precompact-hook
```

The flag writes to `~/.claude/settings.json` via an atomic pattern (tempfile + rename). If the file is malformed, the tool aborts cleanly with a line/column pointer rather than corrupting the configuration.

Cross-cutting options: `--dry-run`, `--user-dir=<path>` (override `HOME` in tests), `--force` to overwrite an existing entry.

See the [full CLI reference](/docs/installation#distill-mcp-setup) for all `setup` flags.

## Verification

```bash
# Confirm the hook is registered
cat ~/.claude/settings.json | jq '.hooks.PreCompact'

# Test the script in isolation
echo '{"event":"PreCompact"}' | bash ~/.claude/hooks/distill-precompact.sh
```

The script output must contain the `preserve verbatim` instruction. No output = something moved upstream — re-run the installer.

## When to use it

Enable the marker if:

- You run long sessions that regularly trigger autocompact.
- Your prompts carry large Distill-compressed outputs (build logs, diffs, AST extracts) where every detail matters.
- You measure the token-to-insight ratio closely and want to capitalise on the initial compression investment.

Leave it off if:

- Your sessions are short and never trigger autocompact.
- You're comfortable with a second summarisation pass over already-compressed content.
- You're testing an existing `v0.8.x` / `v0.9.x` integration and want strictly identical behaviour.
