# Intégration Cursor

Ce guide vous accompagne dans la configuration de Distill avec l'IDE Cursor.

## Prérequis

- Cursor installé ([télécharger](https://cursor.com))
- Node.js 20+ installé

## Configuration rapide (30 secondes)

```bash
npm install -g distill-mcp
distill-mcp setup --cursor
```

Terminé ! Redémarrez Cursor et vous êtes prêt.

## Configuration manuelle

Si la configuration automatique n'a pas fonctionné :

**Via l'interface :** File → Preferences → Cursor Settings → MCP → Add Custom MCP

**Via fichier :** Éditez `~/.cursor/mcp.json` (global) ou `.cursor/mcp.json` (par projet) :

```json
{
  "mcpServers": {
    "distill": {
      "command": "distill-mcp",
      "args": ["serve"]
    }
  }
}
```

Ou avec npx (sans installation globale) :

```json
{
  "mcpServers": {
    "distill": {
      "command": "npx",
      "args": ["distill-mcp", "serve"]
    }
  }
}
```

## Vérifier l'installation

Redémarrez Cursor, puis demandez dans le chat IA :

```
Quels outils MCP sont disponibles ?
```

Vous devriez voir les outils Distill listés.

## Utilisation de Distill avec Cursor

### Navigation dans le code

Utilisez la lecture intelligente de fichiers pour explorer le code :

```
Utilise smart_file_read pour montrer la structure de src/api/routes.ts
```

### Analyse des erreurs de build

Quand les builds échouent :

```
Analyse cette sortie de build avec auto_optimize : [collez les erreurs]
```

### Compression des diffs

Pour les diffs volumineux :

```
Compresse ce git diff : [collez le diff]
```

## Bonnes pratiques

1. **Utilisez smart_file_read** pour l'exploration de code au lieu d'ouvrir les fichiers directement
2. **Compressez les sorties de build** avant de les coller dans le chat
3. **Vérifiez session_stats** périodiquement pour voir vos économies

## Dépannage

### Serveur MCP introuvable

1. Vérifiez l'installation : `npx distill-mcp --version`
2. Consultez les logs MCP de Cursor dans le panneau Output
3. Essayez d'exécuter le serveur manuellement pour vérifier les erreurs

### Les outils ne fonctionnent pas

1. Redémarrez complètement Cursor
2. Vérifiez que la config JSON est valide dans `~/.cursor/mcp.json`
3. Assurez-vous que Node.js 20+ est dans votre PATH
4. Vérifiez les logs dans le panneau Output (View → Output → MCP)
